## What is Vencore?

Vencore is a modular, white-label business platform built for organizations that need software tailored to their own workflows.

Rather than forcing businesses to adapt to rigid, one-size-fits-all solutions, Vencore provides a flexible foundation with authentication, organizations, users, permissions, APIs, navigation, and extensibility built in. Business functionality is delivered through installable plugins, allowing every deployment to be customized without modifying the core platform.

Whether you're building a CRM, ERP, HRMS, Inventory Management System, Helpdesk, or a completely custom business application, Vencore provides the infrastructure so you can focus on solving business problems instead of rebuilding common functionality.

---

## Plugin-First Architecture

Every major feature in Vencore is implemented as a plugin.

Plugins can:

- Add frontend pages
- Register backend APIs
- Create database models and migrations
- Extend navigation
- Register permissions
- Integrate third-party services
- Listen to platform events
- Extend or replace existing functionality

This architecture keeps the platform modular, maintainable, and easy to customize while allowing organizations to install only the features they need.

---

## White-Label by Design

Vencore is built for agencies, enterprises, and SaaS providers that deploy software for multiple organizations.

Each deployment can have its own:

- Branding
- Theme
- Domain
- Enabled plugins
- Authentication providers
- Business workflows
- User permissions

All while running on the same platform.

## Public Repositories

### `vencore-plugin-sdk`

The official SDK for building Vencore plugins.

It provides the APIs, types, utilities, and development tools required to create plugins that integrate seamlessly with the Vencore platform. If you're extending Vencore, this is the repository you'll use.

### `install`

The official installer and bootstrap tool for creating and configuring new Vencore deployments.
